# Phase 2 DSP Survey: Reverb Topologies

Research memo comparing three algorithmic reverb topologies as potential Phase 2 additions to the Premonition offline-render pipeline. Existing code: [Reverb.h](../Premonition/dsp/Reverb.h) (Freeverb — 8 LPFCombs + 4 SchroederAllpasses, per-channel with a 23-sample stereo spread) and [DelayLine.h](../Premonition/dsp/DelayLine.h) (modulo-indexed circular buffer, `tickAndRead` / `peek`). Both primitives are directly reusable.

Generated by Agent 2 during Step 3 of [HANDOFF-2026-04-19-reverb-types-and-modes.md](HANDOFF-2026-04-19-reverb-types-and-modes.md). Purpose: inform a future-vs-now decision. Not an implementation plan.

## 1. Dattorro Plate Reverb (1997)

Jon Dattorro, "Effect Design Part 1: Reverberator and Other Filters," *JAES* 45(9). A figure-eight topology: input diffuser (4 series allpasses) feeds a cross-coupled tank of two half-circuits, each half containing a modulated allpass, a delay, a damping LPF, a fixed allpass, and another delay. Outputs are tapped from specific points in each half and summed with signed coefficients.

### Code sketch

```cpp
struct DattorroPlate {
  SchroederAllpass inDiff[4];            // g = 0.75, 0.75, 0.625, 0.625
  ModulatedAllpass modAP[2];             // g=0.7, D≈672/908 @ 29.8kHz, LFO depth ~8
  DelayLine         delay[4];            // D ≈ 4453, 3720, 4217, 3163 @ 29.8kHz
  DampingLPF        damp[2];             // one-pole, cutoff from "damping"
  SchroederAllpass  fixedAP[2];          // g = 0.5
  float decay = 0.5f, bandwidth = 0.9995f;
  float x1 = 0, x2 = 0;                  // cross-tank state

  float processMono(float x, float& outL, float& outR) {
    // Input conditioning
    float s = inputLPF(x) * bandwidth;
    for (auto& a : inDiff) s = a.process(s);

    // Tank halves (cross-coupled)
    float a = s + x2 * decay;
    a = modAP[0].process(a);
    a = delay[0].process(a);
    a = damp[0].process(a) * decay;
    a = fixedAP[0].process(a);
    x1 = delay[1].process(a);

    float b = s + x1 * decay;
    b = modAP[1].process(b);
    b = delay[2].process(b);
    b = damp[1].process(b) * decay;
    b = fixedAP[1].process(b);
    x2 = delay[3].process(b);

    // Output taps (Dattorro Table 1, signed)
    outL =  0.6f*delay[2].tap(266) + 0.6f*delay[2].tap(2974)
          - 0.6f*fixedAP[1].tap(1913) + 0.6f*delay[3].tap(1996)
          - 0.6f*delay[0].tap(1990) - 0.6f*fixedAP[0].tap(187)
          - 0.6f*delay[1].tap(1066);
    outR = /* mirror taps from the other half */;
    return 0;
  }
};
```

### CPU cost
~1.3–1.5x Freeverb. Fewer total delay lines (≈10 vs. 24 stereo), but multi-tap outputs and modulated allpasses add per-sample work. Dominant cost is still delay line indexing.

### Open-source references
- **freeverb3 / Jezar follow-ups (pkg `freeverb3`)** — LGPL-2.1+. Includes a Dattorro port. **LGPL is a blocker for closed-source static linking** without shim dance; avoid as a copy source.
- **ValleyAudio/ValleyRackFree — Plateau** (github.com/ValleyAudio/ValleyRackFree) — **GPL-3.0. Blocker.** Do not read for porting.
- **Signalsmith "Let's Write a Reverb" article + reference code** (signalsmith-audio.co.uk) — MIT-style permissive (check repo header; historically permissive). **Safe.**
- **mdaPlugins / mda-vst JS Dattorro** (e.g., the Max/MSP `gen~` Dattorro by Stephen Coyle) — permissive/public-domain variants exist; verify per-file header.

### Integration difficulty
**Low-to-moderate.** Our `SchroederAllpass` and `DelayLine` drop in as-is. We need two small additions: (a) a modulated allpass — a `SchroederAllpass` variant whose delay read uses a fractional tap driven by an LFO (Lagrange or linear interp on `DelayLine::peek`); (b) multi-tap support on `DelayLine`, which is trivial (`peek(offset)` already exists). The structural wiring is the meaningful work — Dattorro's exact tap indices and signs must be transcribed faithfully from the paper. Tuning can follow Dattorro's constants verbatim at 29.8 kHz and scale by `sampleRate/29761.0`. Expect 1–2 days implementation, 1 day tuning.

## 2. Schroeder-Moorer / Van Duyne Allpass-Chain Spring Reverb

A long series chain of allpasses — typically 60–200 — with increasing delays and moderate gains (g ≈ 0.6–0.75) creates the dense "chirp" and dispersive quality characteristic of spring tanks. Van Duyne's contribution is using *stretched* allpasses (dispersion allpasses, cascaded first-order allpass sections tuned to produce frequency-dependent group delay) to mimic the physical dispersion of a real spring. A parallel low-pass path plus a short slap delay completes the model.

### Code sketch

```cpp
struct SpringReverb {
  static constexpr int kStages = 96;
  SchroederAllpass chain[kStages];        // g ≈ 0.65, D = primes 30..450
  SchroederAllpass disperse[4];           // short APs, g ≈ 0.5, D = 4..12
                                          //   → frequency-dependent delay
  DelayLine slap;                         // ~30 ms pre-delay
  OnePoleLPF hiCut;                       // ~4 kHz, emulates spring bandwidth
  OnePoleHPF loCut;                       // ~150 Hz, removes rumble
  float feedback = 0.6f;
  float tail = 0.0f;

  void prepare(float fs) {
    for (int i = 0; i < kStages; ++i) {
      int D = primeNear(30 + i * 4 * fs/44100.0f);
      chain[i].prepare(D, (i & 1) ? 0.68f : -0.62f);  // alternate sign for chirp
    }
    for (auto& d : disperse) d.prepare(8, 0.55f);
    slap.setSize(int(0.03f * fs));
  }

  float process(float x) {
    float s = loCut.process(x) + tail * feedback;
    for (auto& ap : disperse) s = ap.process(s);
    for (auto& ap : chain)    s = ap.process(s);
    s = hiCut.process(s);
    tail = s;
    return slap.tickAndRead(s, slap.size() - 1) * 0.7f + s * 0.3f;
  }
};
```

### CPU cost
~2.5–4x Freeverb depending on chain length. 96–200 allpasses dwarfs Freeverb's 12 filters. Offline-only, so fine — but it is the heaviest of the three.

### Open-source references
- **Faust `dm.spring_reverb` / `spring.dsp`** (faustlibraries) — **MIT-equivalent STK-4.3 / permissive**, header on each file. **Safe** — high-quality reference, Van Duyne-flavored.
- **Regalia/Smith "Reverberation" lectures + JOS CCRMA code examples** — **public domain / STK license** per-file. **Safe.**
- **elk-audio / surge-synthesizer Spring** (github.com/surge-synthesizer/surge) — **GPL-3.0. Blocker** as a copy source; reference for architecture only via paper.
- **earlevel spring reverb tutorial** (Nigel Redmon) — permissive snippets, usable with attribution.

### Integration difficulty
**Moderate.** Structurally simpler than Dattorro (long linear chain, no cross-coupling, no modulation), but tuning is the hardest of the three. The character comes from exact delay primes, alternating gain signs, and the dispersion section — wrong constants give a harsh metallic buzz rather than a spring. Our `SchroederAllpass` handles both positive and negative `g` with no changes. Need to add a prime-generator helper and a `OnePoleLPF/HPF` primitive (we have the LPF idiom inside `LPFComb` — lift it out). Expect 1 day implementation, 2–3 days tuning against reference recordings.

## 3. Jot FDN Hall Reverb

Jean-Marc Jot's feedback delay network: N parallel delay lines (typically N=8 or 16) whose outputs are mixed by an NxN unitary feedback matrix (Householder or Hadamard) and re-injected. Per-line damping filters in the loop control frequency-dependent decay. Tonally the most "hall-like" of the three — smooth, colorless tail, very controllable RT60 vs. frequency.

### Code sketch

```cpp
struct JotFDN8 {
  static constexpr int N = 8;
  DelayLine   lines[N];               // mutually prime D: 743,821,941,...,1693
  OnePoleLPF  damping[N];             // per-line, T60 shaping
  float       b[N];                   // input gains
  float       c[N];                   // output gains
  float       state[N] = {};

  // 8x8 Householder: A = I - (2/N) * u * u^T with u = [1,1,...,1]
  // Equivalent fast form: y = x - (sum(x) * 2/N)
  static void mix(float* v) {
    float s = 0; for (int i = 0; i < N; ++i) s += v[i];
    const float k = 2.0f / N * s;
    for (int i = 0; i < N; ++i) v[i] -= k;
  }

  float process(float x) {
    float taps[N], fb[N];
    for (int i = 0; i < N; ++i) {
      taps[i] = lines[i].peek(lines[i].size() - 1);
      fb[i]   = damping[i].process(taps[i]) * gainForRT60(i);
    }
    mix(fb);                           // unitary recirculation
    for (int i = 0; i < N; ++i)
      lines[i].tickAndRead(x * b[i] + fb[i], lines[i].size() - 1);

    float y = 0; for (int i = 0; i < N; ++i) y += c[i] * taps[i];
    return y;
  }
};
```

### CPU cost
~1.1–1.4x Freeverb for N=8. 8 delay lines + 8 one-poles + O(N) matrix. Householder reduces the matrix to 2N ops, not N². For N=16, ~2x Freeverb.

### Open-source references
- **Faust `dm.zita_rev1` / Fons Adriaensen's zita-rev1** — **GPL-3.0 (zita) / MIT-equivalent (Faust port). Zita is a blocker; the Faust port headers are permissive — check per-file.**
- **CCRMA / JOS FDN reference code** — **STK-4.3 / permissive.** Safe.
- **sfizz / dragonfly-reverb** — **GPL/LGPL. Blockers.**
- **geraintluff/jsfx-reverbs** — check per-file; several are MIT.
- **JUCE dsp `juce::Reverb`** is Freeverb, not FDN; **JUCE 7+ license is GPL-or-commercial** so don't read for porting.

### Integration difficulty
**Lowest of the three.** The structure is orthogonal and short: N delay reads, N damping filters, an O(N) matrix, N delay writes. `DelayLine` is used as-is. We already have the damping-filter idiom. The only conceptual addition is the Householder mix (four lines of code). Parameter mapping is also the cleanest: per-band RT60 → per-line damping-filter coefficients, set analytically. Expect 1 day implementation, 1 day tuning. Delay lengths should be mutually prime to avoid flutter; use a small LUT.

## Recommendation context (not a decision)

If Phase 2 picks one: **Jot FDN** is the lowest-risk, best-tonal-return option and maps cleanest onto our primitives. **Dattorro** is the "plate character" win and is well-documented. **Spring** is the most distinctive sound but the hardest to tune and lives behind the largest CPU and licensing-caution footprint. None of the three require architectural changes to `Reverb.h` — all three can live as sibling classes alongside `ReverbChannel`, selected by a mode enum in `applyReverbStereo()`.

**License watch-list:** freeverb3 (LGPL), ValleyRackFree/Plateau (GPL-3), surge (GPL-3), zita-rev1 (GPL-3), dragonfly-reverb (GPL-3), JUCE Reverb (GPL-or-commercial). Faust libraries and CCRMA/STK code are the safe reading grounds across all three topologies.
